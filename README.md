# ActiveWindowMonitor
一个用于实时监视 Windows 活动窗口信息的桌面工具。它能够显示前台窗口的句柄、进程 ID 以及可执行文件路径，并将窗口切换历史记录到日志文件中。程序还提供了焦点保护功能——可以将某些不想要的窗口（如远程桌面、WSL 终端）自动最小化，并尝试恢复上一个正常窗口。

## 主要功能

- **活动窗口监控**  
  每隔一秒自动检测当前前台窗口，并在界面中刷新显示：
  - 窗口句柄（HWND）
  - 进程 ID（PID）
  - 可执行文件完整路径（EXE）

- **焦点保护 / 黑名单**  
  内置黑名单进程（默认包含 `msrdc.exe`、`wsl.exe`），当这些进程的窗口成为前台时，它们会被自动最小化，同时程序会尝试将焦点切换回上一个正常窗口。切换冷却时间为 500 毫秒，防止频繁操作。

- **日志记录**  
  每次前台窗口发生变化时，自动写入日志文件（UTF-8 编码）。日志保存在程序所在目录的 `logs` 子文件夹中，文件名格式为 `ActiveWindow_YYYYMMDD.log`。最多保留最近 30 个日志文件，旧日志会被自动清理。

- **系统托盘驻留**  
  程序默认隐藏到系统托盘，不会显示在任务栏（可通过托盘菜单勾选“显示在任务栏”）。点击托盘图标左键可显示/隐藏主窗口，右键可打开功能菜单。

- **自定义界面**  
  - 可调整字体名称和字体大小（提供常用等宽字体选项，如 Consolas、Courier New 等）。
  - 窗口位置、尺寸以及“显示在任务栏”状态会被自动保存到配置文件。
  - 当 Caps Lock 键开启时，暂停刷新显示，方便用户复制信息。

- **快速定位 EXE 文件**  
  通过托盘菜单中的“在文件夹中定位 EXE”，可以直接打开资源管理器并选中当前活动窗口对应的可执行文件。

- **顶层显示**  
  主窗口默认设置为 `TopMost`，始终位于其他窗口之上，方便实时观察。

## 系统要求

- Windows 操作系统（Windows 7 及以上，需 .NET Framework 4.0 或更高版本）
- 如果使用 .NET Core / .NET 5+，需将项目迁移并解决 API 兼容性问题（当前代码针对 .NET Framework WinForms）

## 使用方法

1. 运行 `ActiveWindowMonitor.exe`，程序窗口随即出现（默认居中显示，大小为 550×250）。
2. 窗口会实时显示当前前台窗口的信息。
3. 将窗口最小化或关闭（点击关闭按钮），程序会隐藏到系统托盘。
4. 右键点击托盘图标，可进行以下操作：
   - **显示窗口** – 重新打开主界面。
   - **显示在任务栏** – 切换是否在 Windows 任务栏显示。
   - **字体名称 / 字体大小** – 调整信息显示字体。
   - **在文件夹中定位 EXE** – 定位当前活动窗口的可执行文件。
   - **保存配置 / 应用配置** – 手动保存或重新加载配置文件。
   - **退出** – 完全退出程序。
5. 如需临时冻结显示（例如复制文本），可以按下 **Caps Lock** 键，此时屏幕刷新将暂停。再次解除 Caps Lock 后恢复更新。

## 配置说明

程序会将设置保存在自身所在目录下的 `ActiveWindowMonitor.ini` 文件中，内容包括：

```
X=100
Y=200
Width=550
Height=250
ShowInTaskbar=False
FontName=Consolas
FontSize=11
```

- **X, Y** – 窗口左上角坐标。
- **Width, Height** – 窗口宽度和高度。
- **ShowInTaskbar** – 是否在任务栏显示（`True` 或 `False`）。
- **FontName, FontSize** – 显示字体名称和大小。

如果配置文件不存在，程序将使用默认设置并在首次关闭时自动生成。  
从旧版本（原先保存在 `%LocalAppData%` 下）迁移时，程序会自动将旧配置文件复制到新位置。

## 编译与运行

### 使用 Visual Studio

1. 将 `Form1.cs` 代码放入新的 Windows Forms 项目（.NET Framework）。
2. 确保项目引用了 `System.Windows.Forms`、`System.Drawing` 等必要程序集。
3. 准备一个图标文件 `ActiveWindowMonitor.exe.ico`（或与最终 exe 同名的 `.ico` 文件），放在输出目录下作为窗体和托盘图标。若无图标，程序将使用默认图标。
4. 编译生成 exe 文件后即可直接运行。

### 命令行编译（csc.exe）

如果安装了 .NET Framework SDK，可以使用以下命令编译：

```
csc /target:winexe /reference:System.dll,System.Drawing.dll,System.Windows.Forms.dll Form1.cs
```

编译完成后，确保将对应的 `.ico` 文件放在 exe 同目录。

## 注意事项

- 程序需要以普通用户权限运行即可获取大部分进程信息，但对于某些受保护的系统进程可能无法读取 EXE 路径，界面会显示“无法获取路径（权限不足）”。
- 黑名单进程列表和切换冷却时间可在代码中直接修改（`blacklistProcessNames` 和 `switchCooldownMs`）。
- 程序自身的进程 ID 会被忽略，当前台窗口是本程序时，界面不会刷新，避免干扰复制操作。
- 如果窗口被拖到屏幕外导致下次启动时不可见，程序会强制将其移动到主屏幕中心。
